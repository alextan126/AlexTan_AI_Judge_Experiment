题目：
某企业级业务管理系统在处理大量用户操作日志时，发现数据库写入成为瓶颈。为了提升系统响应速度，你需要在以下两种存储优化方案中进行二选一决策：A. 使用 MySQL 的 B+ 树索引对日志表进行全量排序并建立多级索引；B. 将实时高频写入的日志数据先存入 Redis List（基于链表实现的队列），再异步批量同步至 MySQL。你会选择哪种方案？请简述理由。


样本测试序号 A-01
STARE_level: STARE / score: 3

模拟候选人回答：
针对大量用户操作日志导致数据库写入成为瓶颈的问题，我选择方案 B。核心矛盾是高频写入压力，而方案 A 建立多级索引不仅无法缓解写入瓶颈，反而因为每次写入都需要维护 B+ 树结构（如页分裂等），会进一步拖慢写入速度。方案 B 引入 Redis List 作为消息队列进行削峰填谷，Redis 基于内存操作，写入性能极高，能迅速响应前端请求；随后通过异步线程将数据批量同步至 MySQL，将大量零散的单条 INSERT 转换为批量写入，大幅降低了数据库的连接开销和磁盘 I/O 频率。这种架构既提升了系统的整体响应速度，又保护了底层关系型数据库不被高并发写垮。

备注：
完美的标准答案。包含了 S/T（转述问题：大量日志导致数据库写入瓶颈），A（选择方案 B），R（方案 B 利用 Redis 缓冲并异步批量写入 MySQL），E（深入分析了方案 A 的反作用（维护索引拖慢写入），并解释了方案 B 为什么有效（内存操作削峰、批量插入减少 I/O））。
