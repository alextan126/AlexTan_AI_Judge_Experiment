## 一、稳定性总结（按题型）

### Concept 题 — 全部 Gini = 0，完全稳定

涵盖 6 个岗位、所有 Item（全对/瑕疵/错误），concept 的每一个判定点（dp: define; ep: keyword, keyword_first_sentence, keyword_no_repeat, not_overtime, define_first_sentence, order_define_explain）在 5 轮测试中均无波动。即使系统判定结果与 GT 不一致（如瑕疵样本中 keyword_first_sentence/keyword_no_repeat 的系统性误判），系统自身的输出也是 5 轮完全一致的。

### Application 题 — 全部 Gini = 0，完全稳定

同样涵盖所有岗位和所有 Item，application 的 dp（correct_option）和 ep（hook_original, deduction_correct）均无波动。通用岗 Item 2 的 correct_option 虽然与 GT 不符（系统=0, GT=1），但 5 轮输出完全一致。

### Code 题 — 仅有的 3 个不稳定点均出现在此

| 岗位 | 样本 | 判定点 | 5轮序列 | Gini |
|------|------|--------|---------|------|
| 算法工程师 | Item 3 错误 (K-05) | syntax_correct | [0,1,1,1,0] | **0.4800** |
| 解决方案 | Item 2 瑕疵 (K-02) | syntax_correct | [0,0,1,0,0] | **0.3200** |
| 解决方案 | Item 2 瑕疵 (K-02) | function_correct | [1,1,1,1,0] | **0.3200** |

**结论**：concept 和 application 两种题型的 Gini 在全部测试中保持 0（100% 稳定），不稳定性完全集中在 code 题型的决定项上。这可能说明：

- concept 和 application 的判定逻辑是确定性的（文本匹配/选项比对），不受 LLM 随机性影响
- code 题型的 syntax_correct 和 function_correct 判定涉及更复杂的代码理解，在边界样本（转义字符、极简代码）上容易受到 LLM 采样随机性的影响，导致轮间波动
